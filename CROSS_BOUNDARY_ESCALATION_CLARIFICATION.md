# Cross-Boundary Cluster Escalation - Clarification Required

## 🚨 SUPERVISOR QUESTION: "Escalation automatica... to WHO?"

### ❌ CURRENT PROBLEMATIC TEXT:
```
Identificazione Cluster Cross-Boundary 

Uno dei rischi principali in sanità pubblica è la gestione frammentata di fenomeni che superano confini amministrativi. 

Il sistema identifica: 
cluster che attraversano confini comunali, 
continuità di hotspot su territori di ASL diverse, 
pattern lineari (es. lungo corridoi infrastrutturali o bacini idrici). 

Criteri di priorità cross-boundary: 
coinvolgimento ≥ 3 comuni contigui, 
appartenenza a più ASL, 
persistenza ≥ 5 giorni, 
coerenza con forecast in aumento. 

In tali casi, il sistema suggerisce escalation automatica.
```

**PROBLEMA**: Non specifica CHI riceve l'escalation!

---

## ✅ OPZIONI DI CHIARIMENTO:

### **Opzione 1 - Escalation Locale (Raccomandato):**
```
Identificazione Cluster Cross-Boundary 

Il sistema identifica cluster che attraversano confini amministrativi:
• Pattern su più comuni contigui
• Hotspot tra diverse ASL  
• Spread lungo infrastrutture o bacini idrici

Criteri di allerta cross-boundary:
• Coinvolgimento ≥ 3 comuni contigui
• Appartenenza a più ASL
• Persistenza ≥ 5 giorni
• Trend in crescita

In tali casi, il sistema genera notifica ai responsabili ASL coinvolti per coordinamento locale.
```

### **Opzione 2 - Nessuna Escalation Automatica:**
```
Identificazione Cluster Cross-Boundary 

Il sistema rileva automaticamente:
• Cluster tra più comuni
• Hotspot su territori ASL diverse
• Pattern lungo corridoi infrastrutturali

Criteri di identificazione:
• ≥ 3 comuni contigui coinvolti
• Multiple ASL interessate  
• Persistenza ≥ 5 giorni
• Forecast in aumento

Il sistema segnala questi pattern nella dashboard per valutazione manuale del personale ASL.
```

### **Opzione 3 - Rimozione Completa (SCELTA FINALE):**
```
Identificazione Cluster Geografici

Il sistema identifica automaticamente:
• Concentrazioni geografiche di casi
• Pattern spaziali di diffusione  
• Hotspot territoriali attivi

I risultati sono visualizzati nella dashboard per supportare le decisioni operative del personale ASL locale.
```

### **Alternativa Ultra-Semplice:**
```
Analisi Geografica

Il sistema mostra sulla mappa:
• Zone con maggiori concentrazioni di casi
• Aree geografiche a rischio elevato
• Distribuzione territoriale delle malattie

Le informazioni sono disponibili nella dashboard per la consultazione del personale ASL.
```

---

## 🎯 RACCOMANDAZIONE PER SUPERVISORE:

**USA OPZIONE 1** perché:
- ✅ **Specifica chiaramente** CHI riceve le notifiche (responsabili ASL)
- ✅ **Mantiene coordinamento locale** senza escalation esterna complessa
- ✅ **Elimina ambiguità** su destinatari delle allerte
- ✅ **Fattibile** con risorse ASL esistenti
- ✅ **Nessuna dipendenza** da enti esterni

---

## 🚫 COSA EVITARE:

❌ **Non specificare destinatari** (problema attuale)
❌ **Escalation a livelli regionali/nazionali** (troppo complesso)  
❌ **Automatismi inter-organizzativi** (non fattibile)
❌ **Coordinamento multi-ASL automatico** (risorse insufficienti)

---

## 💡 SOLUZIONE FINALE:

**Sostituisci la sezione problematica con l'Opzione 1** che chiarisce:
- **CHI**: Responsabili ASL coinvolti
- **COSA**: Notifica per coordinamento
- **COME**: Attraverso dashboard locale
- **PERCHÉ**: Gestione coordinata locale

Questo elimina l'ambiguità e fornisce una soluzione pratica e fattibile!
